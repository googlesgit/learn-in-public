# Review checklist (before you approve)

Use this every time — 5–15 minutes. The graph square is worthless without this step.

## All days

- [ ] I read the full draft and understand it
- [ ] I removed wrong or generic AI fluff
- [ ] I added **one personal line** (what surprised me, what I'd do differently)
- [ ] Title and date are correct
- [ ] Commit message describes the content, not "daily update"

## Day A — Learn

- [ ] Example code runs or is clearly marked pseudo-code
- [ ] I could explain this topic to someone else in 2 minutes
- [ ] Links point to real docs (MDN, official guides, etc.)

## Day B — Case study

- [ ] Framed as **hypothetical** ("If X had to…"), not "I worked at X"
- [ ] Has tradeoffs and non-goals
- [ ] Diagram matches the narrative

## Day C — Audit

- [ ] Target is **my repo / my URL** only (see `config/audit-targets.json`)
- [ ] Findings are verified (not invented)
- [ ] Severity labels make sense (critical / high / medium / low)
